# 代码功能快速定位与技术栈说明

> 目标：让“代码小白”也能按功能快速找到实现位置，并理解关键技术点和第三方库用途。

## 一、项目分层（先看这个）

- `recycling.Web.UI`：表现层（MVC 控制器 + 视图）
  - 你看到的页面、按钮、接口入口主要在这里。
- `recycling.BLL`：业务逻辑层
  - 处理“规则”和“流程”，例如预约校验、订单状态流转。
- `recycling.DAL`：数据访问层
  - 负责 SQL 查询与数据库读写。
- `recycling.Model`：数据模型层
  - 实体类、页面提交模型、返回模型都在这里。
- `recycling.Common`：公共工具层
  - 日志、常量、字符串/时间扩展、校验、邮件等通用能力。

## 三、关键专业技术点（小白必看）

1. **ASP.NET MVC（System.Web.Mvc）**
   - 使用“控制器 Action + 视图 cshtml”模式。
   - 控制器负责接收请求、调用 BLL、返回页面或 JSON。

2. **三层架构（UI/BLL/DAL）**
   - UI 不直接写 SQL；先走 BLL，再走 DAL。
   - 好处：便于维护、复用、定位问题。

3. **ViewModel（页面模型）与 Entity（数据库实体）分离**
   - `recycling.Model` 里既有实体也有页面提交模型。
   - 好处：避免页面输入直接污染数据库实体。

4. **会话机制（Session）**
   - 登录后把用户对象放在 Session 中。
   - 用于识别当前登录人和权限。

5. **服务端校验 + 前端交互校验**
   - 服务端通过模型验证和业务规则校验保证安全。
   - 前端负责提升交互体验，但不能替代服务端校验。

## 四、第三方库/插件说明（含用途）

1. **Newtonsoft.Json（Json.NET）**
   - 位置示例：`recycling.BLL/AppointmentBLL.cs`
   - 用途：把对象序列化为 JSON 字符串，或从 JSON 反序列化为对象。
   - 场景：保存预约品类问答、复杂字段结构化存储。

2. **jQuery（前端脚本库）**
   - 位置示例：`recycling.Web.UI/Views/Shared/_Layout.cshtml` 等布局页
   - 用途：DOM 操作、Ajax 请求、事件处理。

3. **Bootstrap（前端样式组件库）**
   - 位置示例：`recycling.Web.UI/Content/` 与共享布局引用
   - 用途：页面栅格、按钮、表单、弹窗等快速 UI 构建。

> 说明：`System.Web.Mvc`、`System.Data.SqlClient` 等属于 .NET 框架组件，不属于第三方库。

## 五、给“查找功能代码”的实用方法

1. 先定功能（例如“预约提交”）。
2. 在 `Controllers` 找入口 Action（例如 UserController 中的预约相关方法）。
3. 继续看该 Action 调用了哪个 `BLL` 方法。
4. 在对应 `BLL` 中看业务规则，再跳到对应 `DAL` 看数据库读写。
5. 需要改页面展示时，再回到 `Views` 找对应 `.cshtml`。

这样顺序查找，基本不会迷路。
